# 文字化けフォルダ対応方法

## 問題

PowerShellのエンコーディング問題により、以下の文字化けフォルダが存在している可能性があります：

- `01-迺ｰ蠅・ｧ狗ｯ・` - 正しくは `01-環境構築`
- `08-繝・ヰ繝・げ繝ｻ繝医Λ繝悶Ν` - 正しくは `08-デバッグ・トラブル`
- `06-Shopify騾｣謳ｺ` - 正しくは `06-Shopify連携`

## 確認方法

1. エクスプローラーで `docs/05-development` フォルダを開く
2. 文字化けしたフォルダ名があるか確認
3. 正しいフォルダ名と比較

## 対処方法

### 方法1: エクスプローラーで直接確認・削除（推奨）

1. エクスプローラーで `docs/05-development` を開く
2. 文字化けしたフォルダを見つける
3. 中身を確認（正しいフォルダに既に統合されているか確認）
4. 空であれば削除
5. 中身があれば、正しいフォルダに移動してから削除

### 方法2: Git Bashで確認・削除

```bash
# Git Bashを開く
cd docs/05-development

# 文字化けしたフォルダを確認
ls -la | grep "01-\|06-\|08-"

# 文字化けしたフォルダを削除（中身が空の場合）
rm -rf "01-迺ｰ蠅・ｧ狗ｯ・"
rm -rf "08-繝・ヰ繝・げ繝ｻ繝医Λ繝悶Ν"
```

## 現状

- 正しいフォルダは存在している：
  - `01-環境構築\`
  - `06-Shopify連携\`
  - `08-デバッグ・トラブル\`

- 文字化けフォルダの内容は既に正しいフォルダに移動済みの可能性が高い

## 次のステップ

1. エクスプローラーで文字化けフォルダの存在を確認
2. 中身が空なら削除
3. 削除後、`git status`で変更を確認
4. 変更をコミット
