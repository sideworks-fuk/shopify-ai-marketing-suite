# Backlog チケット: UI/UX改善 - 操作の統一

## 件名
【UI/UX】前年同月比・購入回数分析で条件変更時の挙動を統一する

---

## 本文

### 対象画面
- 前年同月比分析画面（商品）
- 購入回数分析画面

### 事象・要望
条件変更時の挙動が画面内・画面間でバラバラで分かりづらい。

- **並び順**変更時: 即時再描画される
- **表示データ**変更時: 「分析実行」を押さないと反映されない
- **対象年**変更時: テーブルタイトルのみ切り替わり、データは再取得されない（分析実行が必要なのに即時更新されていない）

### 対応方針
- データ再取得が必要な条件（対象年・表示データなど）は、変更時に自動で再取得するか、いずれにしても「分析実行」必須で統一する
- 並び順・フィルタなどクライアント側で完結する変更は即時再描画で統一する
- 上記を「条件の種類ごと」にルールを決め、前年同月比分析・購入回数分析の両方で同じ挙動にそろえる

### 完了条件
- 対象2画面で、同じ種類の条件変更に対して同じ挙動になっていること
- 対象年変更時にタイトルだけ変わりデータが古いまま、という状態が解消されていること
