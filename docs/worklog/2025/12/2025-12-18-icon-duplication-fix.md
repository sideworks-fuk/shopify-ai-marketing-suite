## 作業概要
リアルタイム同期と初期設定で同じアイコンが使用されている問題を修正

## 問題の詳細

### 発生状況
- 初期設定とリアルタイム同期の両方で`Zap`アイコンが使用されていた
- 手動同期と自動同期でも同様のアイコン重複があった
- ユーザーが機能を識別しにくい状態

### 影響
- UI/UXの混乱
- 機能の視覚的区別が困難

## 実施内容

### frontend/src/app/setup/initial/page.tsx

#### 1. アイコンのインポート追加
```tsx
import { 
  // ... 既存のインポート
  Settings,  // 追加
  Zap,
  // ...
} from 'lucide-react'
```

#### 2. タブのアイコン変更

**Before:**
- 初期設定: `<Zap />` 
- 同期履歴: `<Clock />`
- 手動同期: `<PlayCircle />`

**After:**
- 初期設定: `<Settings />` ⚙️
- 同期履歴: `<Clock />` 🕐（変更なし）
- 手動同期: `<RefreshCw />` 🔄

#### 3. リアルタイム同期は別の箇所で維持
- リアルタイム同期: `<Zap />` ⚡（継続使用）

## アイコンの使い分け

| 機能 | 新アイコン | 意味 |
|------|------------|------|
| 初期設定 | Settings | 設定・構成を表す |
| 手動同期 | RefreshCw | 更新・リフレッシュを表す |
| リアルタイム同期 | Zap | 即座・高速を表す |
| 同期履歴 | Clock | 時系列・履歴を表す |

## 技術的変更点

### 変更理由
- **Settings**: 初期設定は「設定」作業なので歯車アイコンが適切
- **RefreshCw**: 手動同期は「更新」動作なので回転矢印が適切
- **Zap**: リアルタイム同期の「即座性」を表現するには稲妻が最適

### 他の重複確認
- AlertCircle: エラー表示で複数使用（問題なし - 同じ意味）
- Clock: 時間関連で複数使用（問題なし - 同じ意味）
