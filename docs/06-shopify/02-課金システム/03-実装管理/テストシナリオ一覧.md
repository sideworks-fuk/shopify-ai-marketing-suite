# 課金テストシナリオ一覧（雛形）

作成日: 2025-08-24  
作成者: ERIS（サポート: 福田）

## 前提条件
- テストストア/アプリの準備完了
- Webhook受信可能

## シナリオ
1. 新規インストール→トライアル開始→承認URL遷移
   - 期待: ステータス trial / confirmationUrl 表示
2. トライアル終了→自動課金開始（シミュレーション）
   - 期待: ステータス active / 課金反映
3. プランアップグレード
   - 期待: プランと金額が更新
4. プランダウングレード
   - 期待: 次サイクルから反映
5. キャンセル処理
   - 期待: cancelled へ遷移、機能制限

## 失敗系
- 承認URL未生成 / GraphQLエラー
- Webhook遅延・重複（冪等）
- レート制限

## 記録
- 実施日時、店舗ID、結果、ログ参照先
- ワークログへの記載ルールに準拠
